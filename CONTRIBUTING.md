### Development Process

1. **Fork the Repository**: Create your own fork of the project
2. **Create a Branch**: Use a descriptive branch name (e.g., `feature/add-cost-estimation`, `fix/floor-plan-parsing`)
3. **Make Changes**: Follow our coding standards and best practices
4. **Write Tests**: Ensure your changes are covered by tests
5. **Update Documentation**: Update relevant documentation
6. **Submit a Pull Request**: Provide a clear description of your changes

### Coding Standards

- **Python**: Follow PEP 8 style guide
- **TypeScript/JavaScript**: Follow Airbnb style guide
- **Commits**: Use conventional commit messages (e.g., `feat:`, `fix:`, `docs:`, `test:`)
- **Type Hints**: Use type hints in Python code
- **Documentation**: Add docstrings to all functions and classes

### Testing Requirements

- All new features must include tests
- Maintain or improve code coverage
- Run the full test suite before submitting PR
- Ensure all tests pass

### Code Review Process

- All submissions require review
- Maintainers will provide feedback
- Address review comments promptly
- Be respectful and constructive
